# ERP Frontend Web Application

기업 내부 업무 프로세스를 웹 기반으로 관리하기 위한  
**ERP(Enterprise Resource Planning) 프론트엔드 웹 애플리케이션**입니다.

백엔드 API 구조를 고려하여  
**인증 흐름, 권한 제어, 기준정보–업무 데이터 분리**를 중심으로 설계했습니다.

## 기준정보 관리 – 거래처 · 품목 마스터

### 거래처 등록
<img src="./거래처등록.png" width="900"/>

- 매출/매입/회계 전표에서 공통으로 사용되는 **거래처 마스터 관리 화면**
- 거래처 코드, 상호, 대표자, 연락처, 주소, 거래 구분(매출/매입) 정보 관리
- 기준정보를 분리하여 **중복 입력 방지 및 데이터 일관성 확보**

**포인트**
- 매출·구매·회계 화면에서 거래처 선택 시 자동 연계
- 실무 ERP 구조를 반영한 마스터–트랜잭션 분리 설계

---

### 품목 등록
<img src="./품목등록.png" width="900"/>

- 원자재 / 부자재 / 제품 / 상품 구분이 가능한 **품목 마스터 관리**
- 규격, 단위, 생산공정, 입고단가, VAT 포함 여부 설정
- 판매·구매·재고 관리 화면과 연동되는 핵심 기준정보

**포인트**
- ERP 실무에서 중요한 **품목구분 + 단가 기준 설계**
- 이후 판매/매출 전표에서 품목 선택 시 자동 반영

---

## 영업 관리 – 판매 등록

### 판매 등록
<img src="./판매등록.png" width="900"/>

- 거래처 및 품목을 선택하여 **판매 내역을 입력하는 영업 트랜잭션 화면**
- 수량 × 단가 기반 금액 자동 계산
- 복수 품목 라인 추가 및 합계 금액 실시간 반영

**포인트**
- 품목/거래처 기준정보와 자연스럽게 연동
- 단순 입력이 아닌 **영업 흐름 중심 UI 구성**

---

## 회계 관리 – 매출전표 등록 및 자동 분개

### 매출전표 등록
<img src="./매출전표등록.png" width="900"/>

- 판매 데이터를 기반으로 매출 전표를 생성하는 회계 처리 화면
- 공급가액, 부가세 자동 계산
- 매출계정·입금계정 설정을 통한 **자동 분개 로직 구현**

### 자동 분개 (미리보기)
- 차변: 매출대금(입금/외상)
- 대변: 매출
- 대변: 부가세예수금
- 차변/대변 합계 자동 검증

**포인트**
- 단순 CRUD가 아닌 **회계 흐름을 이해한 ERP 구현**
- 영업 → 회계 데이터 흐름을 하나의 시나리오로 연결
