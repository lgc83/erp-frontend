## 기준정보 및 매출 관리 화면

실제 ERP 업무 흐름을 기준으로  
**거래처 → 품목 → 판매 → 매출전표**로 이어지는  
기본적인 영업·회계 프로세스를 화면 단위로 구현했습니다.

각 화면은 단순 CRUD가 아니라  
**업무 간 데이터 연결과 회계 처리 흐름**을 고려해 설계했습니다.

------------------------------------------------------------------------------------------------------------------------

### 거래처 등록

<img src="./거래처등록.png" alt="거래처 등록 화면" />

- 매입처 / 매출처 / 매입·매출 거래처 구분 관리
- 거래처 코드 기반 마스터 데이터 관리
- 영업·구매·회계 모듈에서 공통 참조되는 기준정보
- 대표자, 연락처, 주소, 이메일 등 기본 정보 관리

👉 이후 판매·매출전표 등록 시 거래처 선택 데이터로 연동

------------------------------------------------------------------------------------------------------------------------

### 품목 등록

<img src="./품목등록.png" alt="품목 등록 화면" />

- 원재료 / 부재료 / 제품 / 반제품 / 상품 구분 관리
- 규격, 단위, 생산공정 정보 등록
- 입고단가 / 출고단가 및 VAT 포함 여부 설정
- 실제 제조·유통 ERP를 기준으로 한 품목 마스터 구조 설계

👉 판매 등록 및 재고·매출 관리 화면과 연계

------------------------------------------------------------------------------------------------------------------------

### 판매 등록

<img src="./판매등록.png" alt="판매 등록 화면" />

- 거래처 선택 후 판매 품목 등록
- 수량 × 단가 기반 금액 자동 계산
- 다중 판매 라인 추가 기능
- 판매 합계 금액 실시간 계산

👉 단순 판매 기록이 아닌, **회계 전표 생성을 위한 원천 데이터 역할**

------------------------------------------------------------------------------------------------------------------------

### 매출전표 등록

<img src="./매출전표등록.png" alt="매출전표 등록 화면" />

- 판매 데이터를 기준으로 매출 전표 생성
- 공급가액, 부가세, 합계 자동 계산
- 부서, 프로젝트, 거래처 정보 반영
- 매출계정 / 입금계정 설정 가능

#### 자동 분개 미리보기
- 차변: 매출대금(입금/외상)
- 대변: 매출
- 대변: 부가세예수금

👉 **차변·대변 합계 자동 검증**으로 회계 정합성 확보

------------------------------------------------------------------------------------------------------------------------

## 설계 포인트

- 실제 ERP 실무 흐름(기준정보 → 거래 → 전표)을 그대로 반영
- 화면 간 데이터 연결을 고려한 구조 설계
- 단순 UI 구현이 아닌 **업무 프로세스 중심 설계**
- 회계 개념(차변/대변, 부가세 분리)을 이해하고 구현

------------------------------------------------------------------------------------------------------------------------

## 포트폴리오 요약

- ERP 기준정보 / 영업 / 회계 흐름을 이해한 프론트엔드 구현
- 실무형 입력 화면과 데이터 연계 경험
- 매출 전표 및 자동 분개 구조를 UI 레벨에서 구현
- 기업 내부 업무 시스템을 가정한 화면 설계 역량
